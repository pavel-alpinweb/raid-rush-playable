# Playable Ads для Raid & Rush

Данный проект выполнен в рамках тестового задания по созданию Playable Ads. В качестве технологической основы я использовал собственный инструментарий [Charlie](https://alpinweb.com/index.php/portfolio-archive/starter-template-for-html5-games/), базирующийся на движке **Phaser**.

Phaser является оптимальным выбором для разработки Playable Ads, так как он работает на нативных JavaScript-библиотеках и, в отличие от тяжеловесных движков (например, Unity), не тянет за собой лишний runtime. Это позволяет минимизировать вес сборки и обеспечить мгновенную загрузку в рекламных сетях.

## Оптимизация:

- **Использование Phaser:** Phaser обеспечивает низкоуровневый доступ к WebGL, что позволяет эффективно рендерить графику и анимации.
- **Быстрая сборка с Vite:** Vite обеспечивает быструю и эффективную сборку проекта, что помогает сократить размер сборки.
- **Создание текстурных атласов:** Для оптимизации загрузки и рендеринга все спрайты упакованы в атласы с помощью TexturePacker.

## Что бы я улучшил в будущем:

Так как данный проект является тестовым заданием и выполнен в сжатые сроки, он представляет собой функциональный прототип. В полноценной версии я бы реализовал следующие улучшения:

- **Полировка анимаций:** Оригинальные ассеты были в 3D, поэтому я использовал Blender для их рендеринга в 2D-последовательности, совместимые с Phaser. На данном этапе камера и параметры рендеринга настроены вручную для основных движений; в финальной версии требуется более тонкая настройка экспорта и сглаживание переходов между состояниями.
- **Механика прогрессии персонажа:** Судя по предоставленным ассетам (три типа брони: от базовой до элитной), проект подразумевает систему прокачки. Я бы добавил механику автоматического улучшения экипировки при достижении определенног оуровня здоровья.
- **Система смены оружия:** Реализация динамической смены оружия в руках персонажа при получении усилений (Power-ups), выпадающих из поверженных врагов и сундуков.
- **Изометрическая проекция:** В референсе используется изометрия. Для ускорения разработки прототипа я использовал Top-down камеру, которая была предустановлена в инструментарии. В будущем проект стоит перевести на полноценную изометрическую сетку для полного соответствия визуальному стилю оригинала.

## Технологический стек:

- [Storybook](https://storybook.js.org) – изолированная разработка и документация UI-компонентов.
- [i18next](https://www.i18next.com) – система локализации.
- [Vue Router](https://router.vuejs.org) – навигация между экранами и уровнями.

## Установка и сборка:

- **Требования:** Node.js версии 20+
- **Установка зависимостей:** `npm i`
- **Запуск режима разработки:** `npm run dev`
- **Сборка проекта:** `npm run build`
- **Просмотр сборки:** `npx serve dist` (позволяет проверить работоспособность перед деплоем на хостинг или itch.io)

## Storybook:

Используется для разработки и тестирования UI-компонентов (HUD, меню, диалоги) в изоляции от игрового цикла.

- **Запуск:** `npm run storybook` (доступен по адресу `http://localhost:6006`)

## Decap CMS:

Инструмент для оперативного изменения игрового баланса и контента через визуальный интерфейс.

- **Запуск в режиме разработки:** `npm run dev:cms`
- **Админ-панель:** `http://localhost:5173/admin/index.html`

В этом режиме Vite работает совместно с локальным бэкендом Decap, позволяя сохранять изменения конфигурации прямо в файлы проекта.

## Структура проекта:

- `src/composition` – высокоуровневая игровая логика (Phaser).
- `src/configs` – константы, настройки баланса и названия событий.
- `src/scenes` – управление игровыми сценами (`Phaser.Scene`).
- `src/screens` – Vue-компоненты игровых экранов.
- `src/store` – хранилища состояния (Pinia).
- `src/ui-components` – переиспользуемые элементы интерфейса.
- `src/utils` – вспомогательные утилиты.
- `src/i18n.js` – конфигурация локализации.
- `.storybook` – настройки среды Storybook.
- `public/admin` – конфигурация и файлы Decap CMS.
- `public/assets` – графические и аудио ресурсы.
- `electron` – точка входа и настройки Electron.
